# Employee Management System

## О проекте

Employee Management System — это backend-ориентированная система управления персоналом с встроенным SPA-клиентом.
Проект автоматизирует базовые HR-процессы: управление сотрудниками, отделами, должностями, пользователями и ролями.

Приложение реализовано как монолит на Spring Boot с REST API и серверной раздачей клиентской части.

## Ключевая функциональность

### Сотрудники
- создание, редактирование и удаление сотрудников;
- фильтрация по зарплате;
- глобальный поиск по связанным данным;
- расширенный поиск сотрудников по вложенным критериям (`departmentName`, `roleName`, `active`);
- пакетные операции, включая асинхронное обновление зарплат.

### Отделы и должности
- полный CRUD для отделов и должностей;
- просмотр сотрудников по отделу и по должности.

### Пользователи и роли
- CRUD для пользователей;
- CRUD для ролей;
- связывание пользователя с профилем сотрудника;
- управление ролями через связь many-to-many.

### SPA-клиент
- единый интерфейс управления сущностями;
- вкладочная навигация (Employees, Departments, Positions, Users, Roles);
- встроенные формы создания/редактирования;
- локальная пагинация, сортировка и фильтрация.

## Предметная область

Система оперирует основными сущностями:
- `Employee`
- `Department`
- `Position`
- `User`
- `Role`

Ключевые связи:
- `Department` 1:N `Employee`
- `Position` 1:N `Employee`
- `User` 1:1 `Employee` (опционально)
- `User` N:M `Role` через `user_roles`

## Архитектурные особенности

- REST API на Spring MVC;
- слой сервисов с бизнес-логикой и валидацией;
- JPA/Hibernate для работы с БД;
- асинхронные операции через `@EnableAsync`;
- кеширование read-операций;
- централизованная обработка ошибок и DTO-модель;
- health endpoint через Spring Boot Actuator.
